## What is SlateDB?

SlateDB is an embedded storage engine built as a [log-structured merge-tree](https://en.wikipedia.org/wiki/Log-structured_merge-tree) (LSM-tree). Unlike traditional LSM-tree storage engines, SlateDB writes data directly to object storage like S3, GCS, Azure Blob Storage, MinIO, and Tigris.

By leveraging object storage, SlateDB provides:

- **Bottomless storage capacity** - No need to provision or manage disk space
- **High durability** - Built-in replication and durability guarantees from object storage
- **Easy replication** - Leverage object storage's native replication features
- **Cost efficiency** - Pay only for what you store with object storage pricing

<Note>
SlateDB trades higher latency and API costs of object storage for the benefits of bottomless capacity and built-in durability.
</Note>

## Quick Start

Get up and running with SlateDB in minutes:

<CodeGroup>

```toml Cargo.toml
[dependencies]
slatedb = "*"
tokio = "*"
```

```go go.mod
require github.com/slatedb/slatedb-go v0.2.0
```

```gradle build.gradle
implementation 'io.slatedb:slatedb:0.1.0'
```

```bash pip
pip install slatedb
```

</CodeGroup>

Here's a simple example in Rust:

```rust example.rs lines
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    // Setup with in-memory object store
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    let db = Db::open("/tmp/my_db", object_store).await?;

    // Put a key-value pair
    db.put(b"hello", b"world").await?;

    // Get the value
    let value = db.get(b"hello").await?;
    assert_eq!(value, Some("world".into()));

    // Delete the key
    db.delete(b"hello").await?;

    db.close().await?;
    Ok(())
}
```

## Architecture Highlights

SlateDB's architecture is designed for cloud-native workloads:

- **Batched Writes** - MemTables are flushed periodically to object storage as SSTs to reduce PUT API costs
- **Configurable Durability** - Choose between synchronous durable writes or faster asynchronous writes
- **Smart Caching** - Multi-level caching with in-memory block cache, compression, bloom filters, and local disk cache
- **Write-Ahead Log (WAL)** - Optional WAL support for durability guarantees
- **Background Compaction** - Automatic SST compaction to optimize read performance and storage efficiency

## Use Cases

SlateDB is ideal for:

- **Embedded databases** - Applications needing local storage with cloud backup
- **Serverless applications** - No disk management required, scales automatically
- **Multi-region systems** - Leverage object storage replication for global availability  
- **Time-series data** - Efficient writes and range queries for temporal data
- **Caching layers** - Durable cache with object storage backing
- **Data lakes** - Structured storage on top of object storage
