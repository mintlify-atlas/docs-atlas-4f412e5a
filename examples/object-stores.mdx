SlateDB stores all data in an object store backend. This guide shows you how to configure different cloud storage providers.

## Amazon S3

Configure SlateDB to use Amazon S3 or S3-compatible storage (like LocalStack, MinIO):

```rust
use object_store::aws::S3ConditionalPut;
use slatedb::Db;
use std::sync::Arc;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let object_store = Arc::new(
        object_store::aws::AmazonS3Builder::new()
            .with_bucket_name("slatedb")
            .with_region("us-east-1")
            .with_access_key_id("your-access-key")
            .with_secret_access_key("your-secret-key")
            .with_conditional_put(S3ConditionalPut::ETagMatch)
            .build()?,
    );

    let db = Db::open("/tmp/slatedb_s3", object_store).await?;

    // Use the database
    db.put(b"key", b"value").await?;
    println!("Value: {:?}", db.get(b"key").await?);

    db.close().await?;

    Ok(())
}
```

<Note>
Always use `S3ConditionalPut::ETagMatch` for data safety. This prevents concurrent writes from corrupting your database.
</Note>

### S3-Compatible Storage (LocalStack)

For local development with LocalStack or MinIO:

```rust
let object_store = Arc::new(
    object_store::aws::AmazonS3Builder::new()
        .with_allow_http(true)  // Allow HTTP for local development
        .with_endpoint("http://localhost:4566")  // LocalStack endpoint
        .with_access_key_id("test")
        .with_secret_access_key("test")
        .with_bucket_name("slatedb")
        .with_region("us-east-1")
        .with_conditional_put(S3ConditionalPut::ETagMatch)
        .build()?,
);

let db = Db::open("/tmp/slatedb_s3_compatible", object_store.clone()).await?;

// Call db.put with a key and a 64 meg value to trigger L0 SST flush
let value: Vec<u8> = vec![0; 64 * 1024 * 1024];
db.put(b"k1", value.as_slice()).await?;
db.close().await?;
```

<Warning>
Never use `with_allow_http(true)` in production. This is only for local development and testing.
</Warning>

## Google Cloud Storage

Configure SlateDB to use Google Cloud Storage:

```rust
use object_store::gcp::GoogleCloudStorageBuilder;
use slatedb::{config::PutOptions, Db};
use std::sync::Arc;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    // Construct Google Cloud Storage object store
    let gcs_store = Arc::new(
        GoogleCloudStorageBuilder::new()
            .with_service_account_path("/path/to/service-account.json")
            .with_bucket_name("my-slatedb-bucket")
            .build()?,
    );

    println!("Opening the db");
    let path = "/tmp/slatedb_google_cloud_storage";
    let db = Db::open(path, gcs_store.clone()).await?;

    // Put a value and wait for the flush
    println!("Writing a value and waiting for flush");
    db.put(b"k1", b"value1").await?;
    println!("{:?}", db.get(b"k1").await?);

    // Put 1000 keys, do not wait for it to be durable
    println!("Writing 1000 keys without waiting for flush");
    let write_options = slatedb::config::WriteOptions {
        await_durable: false,
    };
    for i in 0..1000 {
        db.put_with_options(
            format!("key{}", i).as_bytes(),
            format!("value{}", i).as_bytes(),
            &PutOptions::default(),
            &write_options,
        )
        .await?;
    }

    // Flush to make the writes durable
    println!("Flushing the writes and closing the db");
    db.flush().await?;
    db.close().await?;

    // Reopen the db and read the value
    println!("Reopening the db");
    let db_reopened = Db::open(path, gcs_store).await?;
    println!("Reading the value from the reopened db");

    // Read 20 keys
    for i in 0..20 {
        println!(
            "{:?}",
            db_reopened.get(format!("key{}", i).as_bytes()).await
        );
    }
    db_reopened.close().await?;

    Ok(())
}
```

<Steps>
  <Step title="Create a service account">
    In Google Cloud Console, create a service account with Storage Object Admin permissions.
  </Step>

  <Step title="Download credentials">
    Download the service account key as a JSON file.
  </Step>

  <Step title="Configure the builder">
    Use `with_service_account_path()` to point to your credentials file.
  </Step>
</Steps>

## Azure Blob Storage

    let blob_store = Arc::new(
        MicrosoftAzureBuilder::new()

            .with_access_key("your-access-key")

## Local File System

For testing or single-machine deployments, use the local file system:

```rust
use slatedb::object_store::local::LocalFileSystem;
use slatedb::Db;
use std::sync::Arc;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let local_root = std::env::temp_dir().join("slatedb-local");
    std::fs::create_dir_all(&local_root)?;
    let object_store = Arc::new(LocalFileSystem::new_with_prefix(local_root)?);

    let db = Db::open("db", object_store).await?;
    db.put(b"hello", b"world").await?;
    db.close().await?;

    Ok(())
}
```

## Write Options

Control write durability with `WriteOptions`:

```rust
use slatedb::config::{PutOptions, WriteOptions};

// Wait for writes to be durable (default)
db.put(b"key", b"value").await?;

// Don't wait for durability (faster, but less safe)
let write_options = WriteOptions {
    await_durable: false,
};
db.put_with_options(
    b"key",
    b"value",
    &PutOptions::default(),
    &write_options,
).await?;

// Manually flush when ready
db.flush().await?;
```

<Note>
**When to use `await_durable: false`:**
- Bulk loading data
- High-throughput scenarios where you can tolerate some data loss
- When you'll call `flush()` yourself at appropriate intervals

**Always use `await_durable: true` (default) when:**
- Data integrity is critical
- You need guaranteed durability for each write
- You're unsure about the trade-offs
</Note>

## Comparison

| Object Store | Use Case | Authentication |
|--------------|----------|----------------|
| **S3** | Production AWS deployments | Access key + secret or IAM roles |
| **GCS** | Production GCP deployments | Service account JSON |
| **Azure Blob** | Production Azure deployments | Account + access key |
| **LocalFileSystem** | Local development, testing | File system permissions |
| **InMemory** | Unit tests, ephemeral data | None |
