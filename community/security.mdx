The security of SlateDB is a top priority. We appreciate your efforts to responsibly disclose any security vulnerabilities you find.

## Supported Versions

We provide security updates for all versions of SlateDB:

| Version | Supported          |
| ------- | ------------------ |
| *       | ✅ Yes             |

<Info>
  All versions of SlateDB currently receive security updates.
</Info>

## Reporting a Vulnerability

If you discover a security vulnerability in SlateDB, please report it responsibly.

<Warning>
  **Do not** open a public GitHub issue for security vulnerabilities. This could put users at risk.
</Warning>

### How to Report

Please report security vulnerabilities through GitHub's security advisory feature:

<Card title="Report a Vulnerability" icon="shield-halved" href="https://github.com/slatedb/slatedb/security/advisories">
  Submit a private security advisory on GitHub
</Card>

### What to Include

When reporting a vulnerability, please include:

<AccordionGroup>
  <Accordion title="Description" icon="file-lines">
    A clear description of the vulnerability and its potential impact.
  </Accordion>
  
  <Accordion title="Steps to Reproduce" icon="list-ol">
    Detailed steps to reproduce the vulnerability, including any necessary code samples or configuration.
  </Accordion>
  
  <Accordion title="Affected Versions" icon="code-branch">
    Which versions of SlateDB are affected by this vulnerability.
  </Accordion>
  
  <Accordion title="Suggested Fix" icon="wrench">
    If you have ideas for how to fix the vulnerability, please include them (optional).
  </Accordion>
</AccordionGroup>

## What to Expect

After you submit a security vulnerability report:

<Steps>
  <Step title="Acknowledgment">
    We will acknowledge receipt of your vulnerability report and begin investigating the issue.
  </Step>
  
  <Step title="Investigation">
    We will investigate the vulnerability and determine its severity and impact.
  </Step>
  
  <Step title="Resolution">
    We will work on a fix and coordinate with you on the disclosure timeline.
  </Step>
  
  <Step title="Disclosure">
    Once a fix is available, we will coordinate the public disclosure of the vulnerability.
  </Step>
</Steps>

<Tip>
  We appreciate your patience during the investigation and resolution process. We are committed to keeping SlateDB secure.
</Tip>

## Security Best Practices

When using SlateDB in production, we recommend:

<CardGroup cols={2}>
  <Card title="Keep Updated" icon="arrow-up">
    Regularly update to the latest version of SlateDB to receive security patches.
  </Card>
  
  <Card title="Monitor Advisories" icon="bell">
    Watch the [SlateDB repository](https://github.com/slatedb/slatedb) for security advisories.
  </Card>
  
  <Card title="Secure Configuration" icon="lock">
    Follow security best practices for your cloud storage provider and infrastructure.

  <Card title="Access Control" icon="key">

  </Card>
</CardGroup>
