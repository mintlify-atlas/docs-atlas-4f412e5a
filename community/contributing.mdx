## Making Substantial Changes

If you plan on making a substantial change to SlateDB, please follow these steps:

<Steps>
  <Step title="Open a GitHub Issue">
    Start by opening a [GitHub issue](https://github.com/slatedb/slatedb/issues) to discuss your proposed changes with the community.
  </Step>
  
  <Step title="Write an RFC (if requested)">
    For significant architectural changes, you may be asked to write an RFC. Use the [0000-template.md](https://github.com/slatedb/slatedb/blob/main/rfcs/0000-template.md) as a template.
  </Step>
  
  <Step title="Submit Incremental PRs">
    Break your work into smaller, incremental pull requests that add functionality while moving toward the goal of the RFC or issue.
  </Step>
</Steps>

<Warning>
  **Large PRs are not accepted.** If you feel the need to submit a large PR, submit it as a draft with a bullet-point list of steps to break it into smaller pieces.
</Warning>

## Pull Request Guidelines

When submitting a pull request, please follow the instructions in the [pull request template](https://github.com/slatedb/slatedb/blob/main/.github/PULL_REQUEST_TEMPLATE.md).

<Tip>
  Keep your PRs focused and small. This makes them easier to review and merge.
</Tip>

## Contributor License Agreement

Most contributions require you to agree to a Contributor License Agreement (CLA) declaring that you have the right to, and actually do, grant us the rights to use your contribution.

When you submit a pull request, a CLA bot will automatically determine whether you need to provide a CLA and decorate the PR appropriately (e.g., status check, comment). Simply follow the instructions provided by the bot.

<Note>
  You will only need to sign the CLA once across all SlateDB repositories.
</Note>

The CLA text is available [here](https://gist.github.com/criccomini/5ea2c60fe2ec791d91aa05d165060b9c).

## AI-Generated Pull Requests

AI-generated (or largely generated) pull requests are welcome, provided that you:

<AccordionGroup>
  <Accordion title="Disclosure Requirements">
    - Disclose that the pull request is largely AI-generated
    - Specify which tool and model you used
  </Accordion>
  
  <Accordion title="Understanding & Quality">
    - Understand the changes in the PR
    - Review the code change yourself before submitting the PR
    - Run `/review` locally in the `codex` or `claude` CLI before submitting the PR
  </Accordion>
  
  <Accordion title="Human Communication">
    - Keep PR descriptions and review replies human-written
    - Follow the guidelines in the PR template (the checkboxes at the bottom)
  </Accordion>
</AccordionGroup>
