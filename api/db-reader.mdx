---
title: 'DbReader'
description: 'Read-only interface for accessing database data'
---

The `DbReader` struct provides a read-only interface for accessing database data, either from the latest state or from a specific checkpoint.

## Opening a Reader

### open

```rust
pub async fn open<P: Into<Path>>(
    path: P,
    object_store: Arc<dyn ObjectStore>,
    checkpoint_id: Option<Uuid>,
    options: DbReaderOptions,
) -> Result<Self, Error>
```

Open a database reader.

<ParamField path="path" type="P: Into<Path>" required>
  The path to the database
</ParamField>

<ParamField path="object_store" type="Arc<dyn ObjectStore>" required>
  The object store to use
</ParamField>

<ParamField path="checkpoint_id" type="Option<Uuid>">
  Optional checkpoint ID to read from. If `None`, creates a new checkpoint from current state
</ParamField>

<ParamField path="options" type="DbReaderOptions" required>
  Configuration options for the reader
</ParamField>

<ResponseField name="Result" type="Result<DbReader, Error>">
  Returns a `DbReader` instance on success
</ResponseField>

```rust
use slatedb::{DbReader, config::DbReaderOptions};

let reader = DbReader::open(
    "test_db",
    object_store,
    None,
    DbReaderOptions::default(),
).await?;
```

### builder

```rust
pub fn builder<P: Into<Path>>(

```

Create a builder for configuring a database reader.

<ParamField path="path" type="P: Into<Path>" required>

</ParamField>

<ParamField path="object_store" type="Arc<dyn ObjectStore>" required>
  The object store to use
</ParamField>

<ResponseField name="DbReaderBuilder" type="DbReaderBuilder<P>">

</ResponseField>

```rust
let reader = DbReader::builder("test_db", object_store)
    .build()
    .await?;
```

<ParamField path="key" type="K: AsRef<[u8]> + Send" required>

</ParamField>

<ResponseField name="Result" type="Result<Option<Bytes>, Error>">

```rust
    K: AsRef<[u8]> + Send,
    T: RangeBounds<K> + Send,
```

Scan a range of keys.

<ParamField path="range" type="T: RangeBounds<K>" required>

<ResponseField name="Result" type="Result<DbIterator, Error>">

## Usage Examples

### Basic Reader Usage

```rust
use slatedb::{Db, DbReader, config::DbReaderOptions, Error};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    // First create and populate a database
    let db = Db::open("test_db", Arc::clone(&object_store)).await?;
    db.put(b"key", b"value").await?;
    db.flush().await?;
    
    // Open a reader
    let reader = DbReader::open(
        "test_db",
        object_store,
        None,
        DbReaderOptions::default(),
    ).await?;
    
    // Read data
    assert_eq!(reader.get(b"key").await?, Some("value".into()));
    
    reader.close().await?;
    Ok(())
}
```

### Reading from a Checkpoint

```rust
// Create a checkpoint in the main database
let checkpoint = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions::default()
).await?;

// Open a reader at that checkpoint
let reader = DbReader::open(
    "test_db",
    object_store,
    Some(checkpoint.id),
    DbReaderOptions::default(),
).await?;

// Reader sees data as it was at checkpoint time
let value = reader.get(b"key").await?;
```

### Scanning with a Reader

```rust
let reader = DbReader::open(
    "test_db",
    object_store,
    None,
    DbReaderOptions::default(),
).await?;

let mut iter = reader.scan(..).await?;
while let Some(kv) = iter.next().await? {
    println!("Key: {:?}, Value: {:?}", kv.key, kv.value);
}
```

### Read-Only Analytics

```rust
use slatedb::config::DbReaderOptions;
use std::time::Duration;

// Create a reader for analytics workload
let reader = DbReader::builder("test_db", object_store)
    .with_options(DbReaderOptions {
        manifest_poll_interval: Duration::from_secs(5),
        checkpoint_lifetime: Duration::from_secs(3600),
        ..Default::default()
    })
    .build()
    .await?;

// Perform analytics queries
let mut iter = reader.scan_prefix(b"metrics:").await?;
let mut total = 0;
let mut count = 0;

while let Some(kv) = iter.next().await? {
    if let Ok(value) = String::from_utf8_lossy(&kv.value).parse::<i64>() {
        total += value;
        count += 1;
    }
}

if count > 0 {
    println!("Average: {}", total / count);
}

reader.close().await?;
```

<Info>
Readers are ideal for analytics workloads as they don't interfere with write operations on the main database.
</Info>

<Note>
When no checkpoint ID is provided, the reader automatically creates and manages a checkpoint that is refreshed periodically according to the `manifest_poll_interval`.
</Note>
